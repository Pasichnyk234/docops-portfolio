# Reference: Documentation content types

## Procedure (How-To)
A step-by-step instruction that guides a user to complete a task.

**Use when:** the user needs to *do* something.  
**Structure:** goal → prerequisites → steps → result.

## Reference
Stable information users look up quickly.

**Use when:** definitions, parameters, interfaces, limits.  
**Structure:** short intro → sections/tables → examples.

## Troubleshooting
Content for solving problems.

**Use when:** a user reports an issue.  
**Structure:** symptom → possible causes → fixes → prevention.
