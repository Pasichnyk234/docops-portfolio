# How-To: Create and publish a MkDocs portfolio site

## Goal
Publish a documentation portfolio as a website using MkDocs and GitHub Pages.

## Prerequisites
- GitHub account
- Repository with a `docs/` folder and `mkdocs.yml`
- GitHub Actions enabled

## Steps
1. Edit content in `docs/` (Markdown files).
2. Commit and push changes to the `main` branch.
3. GitHub Actions deploys the site to the `gh-pages` branch.
4. Open the live site URL in GitHub Pages settings.

## Result
Your documentation is updated online automatically after each push.
